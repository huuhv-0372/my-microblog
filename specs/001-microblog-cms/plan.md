## Summary

Build a small CMS for a microblog where members write short posts (draft в†’ pending в†’ published), assign tags, and readers can comment (anonymous or member). An admin backend moderates both posts and comments before they are publicly visible. The homepage shows a 3-column layout: recent post list (left), paginated post cards (centre), tag cloud with post counts (right). Technical approach: **NestJS monolith with Handlebars SSR**, **Tailwind CSS** compiled to static CSS, **MySQL 8** via **TypeORM**, **JWT in HTTP-only cookies** for auth.

## Technical Context

**Language/Version**: TypeScript 5.x вЂ” `strict: true` enforced via `tsconfig.json`
**Runtime**: Node.js 20 LTS
**Framework**: NestJS (latest stable) with `@nestjs/platform-express`
**View Engine**: Handlebars (HBS) via `express-handlebars` вЂ” SSR, no SPA
**CSS**: Tailwind CSS CLI в†’ compiled to `public/css/tailwind.css`
**Primary Dependencies**:
- `@nestjs/typeorm` + `typeorm` + `mysql2` вЂ” data persistence
- `passport` + `passport-jwt` + `@nestjs/passport` вЂ” authentication
- `bcrypt` вЂ” password hashing (cost factor 12)
- `class-validator` + `class-transformer` вЂ” DTO validation
- `slugify` вЂ” slug generation
- `csurf` вЂ” CSRF protection on mutating forms
- `method-override` вЂ” allow PATCH/DELETE from HTML forms
- `express-handlebars` вЂ” Handlebars layout + partials support

**Storage**: MySQL 8.0+ via TypeORM; 8 migrations (see `data-model.md`)
**Testing**: Jest (unit), Supertest (e2e/integration)
**Target Platform**: Linux server (production), local Node.js (development), Docker Compose (local DB)
**Project Type**: Web application (SSR monolith)
**Performance Goals**: List endpoints <= 300 ms p95; homepage <= 2 s for up to 10,000 posts
**Constraints**: Paginated lists (max 20/page); JWT expiry <= 24 h (access 15 min + refresh 7 d); CSRF on all mutations
**Scale/Scope**: Single-tenant; ~10k posts, ~50k comments in production target

## Complexity Tracking

> No constitution violations requiring justification. Items below are necessary, not speculative.

| Item | Justification |
|------|--------------|
| method-override middleware | HTML forms only support GET/POST; PATCH/DELETE semantics required for post edit/delete. Standard SSR pattern, not speculative. |
| csurf CSRF middleware | Required by Principle IV (Security). All mutating HTML form routes must be CSRF-protected. |
| RefreshToken entity | Required by Principle IV (JWT rotation). Without it, stolen access tokens cannot be invalidated before expiry. |
